Detailed Description

Manages pageview events and analytics data for Parsely on iOS.

Accessed as a singleton. Maintains a queue of pageview events in memory and periodically flushes the queue to the Parsely pixel server.

Method Documentation

- (void) flush

Generate pixel requests from the queue.

Empties the entire queue and sends the appropriate pixel requests. If shouldBatchRequests is YES, the queue is sent as a minimum number of requests. Called automatically after a number of seconds determined by flushInterval.

- (void) flushEvent: (NSDictionary *)  event

Send a single pixel request.

Sends a single request directly to Parsely's pixel server, bypassing the proxy. Prefer sendBatchRequest: to this method, as sendBatchRequest: causes less battery usage

Parameters
eventA dictionary containing data for a single pageview event

- (void) start

Allow the SDK to send pageview events.

Instantiates the callback timer responsible for flushing the events queue. Can be called before of after stop, but has no effect is used before instantiating the singleton

- (void) stop

Disallow the SDK from sending pageview events.
